CSS Class และ Selectors เบื้องต้น
เรียนรู้การใช้ Class และ Selectors ต่างๆ เพื่อเลือกและตกแต่ง HTML Elements อย่างแม่นยำ
CSS Selectors คือเครื่องมือที่ใช้ เลือก HTML Elements ที่เราต้องการตกแต่ง Class และ ID เป็น Selectors ที่สำคัญที่สุด เพราะช่วยให้เราสามารถ เลือกเป้าหมายได้อย่างแม่นยำ การเข้าใจ Selectors ทำให้เราสามารถสร้าง CSS ที่ มีประสิทธิภาพ จัดระเบียบง่าย และบำรุงรักษาสะดวก Selectors เป็นหัวใจสำคัญของการเขียน CSS ที่ดี

CSS Class - การใช้งานพื้นฐาน

Class คือ attribute ใน HTML ที่ใช้จัดกลุ่ม Elements เพื่อให้สามารถตกแต่งพร้อมกันได้ ใน HTML: <div class="highlight">เนื้อหา</div> ใน CSS: .highlight { background-color: yellow; }
HTML CSS ผลลัพธ์
<p class="red-text">ข้อความสีแดง</p> .red-text { color: red; } ข้อความจะเป็นสีแดง
<div class="box">กล่อง</div> .box { border: 1px solid black; } กล่องจะมีเส้นขอบสีดำ
<h2 class="title">หัวเรื่อง</h2> .title { font-size: 24px; } หัวเรื่องจะมีขนาด 24px
ข้อดีของ Class: - ใช้ได้หลายครั้งในหน้าเดียว - จัดกลุ่ม Elements ที่มีลักษณะเดียวกัน - สามารถใช้หลาย Class ใน Element เดียวได้

CSS ID - การใช้งานเฉพาะ

ID คือ attribute ที่ใช้ระบุ Element ใดเฉพาะ ห้ามซ้ำกันในหน้าเดียว ใน HTML: <div id="header">ส่วนหัว</div> ใน CSS: #header { background-color: blue; }
HTML CSS หมายเหตุ
<div id="main-content">เนื้อหาหลัก</div> #main-content { width: 80%; } ใช้แค่ครั้งเดียวในหน้า
<nav id="navigation">เมนู</nav> #navigation { position: fixed; } เหมาะสำหรับส่วนสำคัญ
<footer id="page-footer">ส่วนท้าย</footer> #page-footer { margin-top: 50px; } ID มีความสำคัญสูงกว่า Class
ข้อแตกต่าง Class vs ID: - Class: ใช้ได้หลายครั้ง เขียนด้วย . (จุด) - ID: ใช้ได้แค่ครั้งเดียว เขียนด้วย # (hash) มี Specificity สูงกว่า
CSS Class และ ID Selectors

Basic Selectors ประเภทต่างๆ

ประเภท Selector รูปแบบ ตัวอย่าง การใช้งาน
Universal Selector * * { margin: 0; } เลือกทุก Element
Element Selector tagname p { color: black; } เลือกตาม HTML Tag
Class Selector .classname .button { padding: 10px; } เลือกตาม class attribute
ID Selector #idname #logo { width: 100px; } เลือกตาม id attribute
Group Selector selector1, selector2 h1, h2 { font-weight: bold; } เลือกหลาย Element พร้อมกัน

Descendant Selectors (ตัวเลือกลูกหลาน)

ใช้เลือก Elements ที่อยู่ภายใน Element อื่น (ไม่ว่าจะลึกแค่ไหน) รูปแบบ: parent descendant เขียนด้วย: เว้นวรรคระหว่าง Selectors
CSS Selector HTML ที่ตรงกัน คำอธิบาย
div p { color: red; } <div><p>ข้อความ</p></div> p ที่อยู่ใน div ใดๆ
.container img { width: 100%; } <div class="container"><img></div> img ที่อยู่ใน element ที่มี class container
nav a { text-decoration: none; } <nav><ul><li><a>ลิงก์</a></li></ul></nav> a ที่อยู่ใน nav (ลึกแค่ไหนก็ได้)

Child Selectors (ตัวเลือกลูกตรง)

ใช้เลือก Elements ที่เป็นลูกตรง (direct child) เท่านั้น รูปแบบ: parent > child เขียนด้วย: เครื่องหมาย > ระหว่าง Selectors
CSS Selector HTML ที่ตรงกัน HTML ที่ไม่ตรงกัน
ul > li { list-style: none; } <ul><li>รายการ</li></ul> <ul><div><li>รายการ</li></div></ul>
.menu > a { font-weight: bold; } <div class="menu"><a>ลิงก์</a></div> <div class="menu"><p><a>ลิงก์</a></p></div>

Pseudo Selectors (ตัวเลือกเสมือน)

ใช้เลือก Elements ตามสถานะหรือตำแหน่งพิเศษ Pseudo-classes: ตามสถานะ (:hover, :active, :focus) Pseudo-elements: ตามส่วนประกอบ (::before, ::after)
Pseudo Selector ตัวอย่าง คำอธิบาย
:hover a:hover { color: red; } เมื่อเมาส์วางบนลิงก์
:active button:active { background: gray; } เมื่อกำลังคลิกปุ่ม
:focus input:focus { border: 2px solid blue; } เมื่อ focus ที่ input field
:first-child li:first-child { font-weight: bold; } รายการแรกในลิสต์
:nth-child(n) tr:nth-child(2n) { background: lightgray; } แถวคู่ในตาราง

Attribute Selectors (ตัวเลือกตาม Attribute)

ใช้เลือก Elements ตาม attributes ที่มี
รูปแบบ ตัวอย่าง คำอธิบาย
[attribute] input[required] { border: 1px solid red; } input ที่มี attribute required
[attribute="value"] input[type="email"] { background: lightblue; } input ที่ type เป็น email
[attribute^="value"] a[href^="https"] { color: green; } ลิงก์ที่ขึ้นต้นด้วย https
[attribute$="value"] a[href$=".pdf"] { font-weight: bold; } ลิงก์ที่ลงท้ายด้วย .pdf

การตั้งชื่อ Class ที่ดี

หลักการตั้งชื่อ: - ใช้ชื่อที่อธิบายความหมายมากกว่าลักษณะ - ใช้ kebab-case (คำต่อด้วย -) - หลีกเลี่ยงชื่อที่เฉพาะเจาะจงเกินไป - ใช้ชื่อที่สั้น แต่เข้าใจง่าย
❌ ไม่ควรใช้ ✅ ควรใช้ เหตุผล
class="red-text" class="error-message" อธิบายความหมายมากกว่าสี
class="big_button" class="primary-button" ใช้ kebab-case และความหมาย
class="div1" class="sidebar" ชื่อที่มีความหมายชัดเจน
class="very-very-long-class-name-for-button" class="btn-submit" สั้น กระชับ เข้าใจง่าย

การใช้หลาย Classes

Element เดียวสามารถมีได้หลาย Classes โดยคั่นด้วยเว้นวรรค HTML: <div class="box red large">กล่องสีแดงขนาดใหญ่</div> CSS: .box { border: 1px solid; } .red { color: red; } .large { font-size: 20px; }
HTML CSS ที่ใช้งานได้ ผลลัพธ์
<p class="text highlight"> .text { font-size: 16px; } .highlight { background: yellow; } ข้อความขนาด 16px พื้นหลังเหลือง
<button class="btn primary large"> .btn { padding: 10px; } .primary { background: blue; } .large { font-size: 18px; } ปุ่มสีน้ำเงิน ขนาดใหญ่ มี padding

Specificity (ลำดับความสำคัญ)

เมื่อมี CSS Rules หลายตัวชี้ไปยัง Element เดียวกัน Browser จะใช้ Rule ที่มี Specificity สูงสุด
Selector Specificity Score ตัวอย่าง
Inline style 1000 style="color: red;"
ID 100 #header { color: blue; }
Class, Attribute, Pseudo-class 10 .highlight, [type="text"], :hover
Element, Pseudo-element 1 p, ::before
การคำนวณ: - #header .nav a:hover = 100 + 10 + 1 + 10 = 121 - .menu li a = 10 + 1 + 1 = 12 - Rule แรกจะชนะ เพราะมี Specificity สูงกว่า

Best Practices

แนวทางปฏิบัติที่ดี: - ใช้ Class มากกว่า ID เพื่อความยืดหยุ่น - หลีกเลี่ยง !important ถ้าไม่จำเป็น - เขียน CSS ที่อ่านง่าย และมี comment อธิบาย - จัดกลุ่ม Selectors ที่มี properties เดียวกัน - ใช้ Specificity ต่ำ เพื่อให้แก้ไขง่าย - ตั้งชื่อ Class ให้สื่อความหมาย ตัวอย่างการเขียนที่ดี: /* Navigation styles */ .nav { list-style: none; } .nav-item { display: inline-block; } .nav-link { text-decoration: none; } .nav-link:hover { color: #007bff; }
