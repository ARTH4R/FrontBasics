แอนิเมชันและเอฟเฟกต์ CSS
เพิ่มชีวิตชีวาให้เว็บไซต์ด้วยแอนิเมชันและเอฟเฟกต์ที่สวยงาม
CSS Animation คือเทคนิคที่ใช้ เพิ่มการเคลื่อนไหวและเอฟเฟกต์ ให้กับเว็บไซต์ ช่วยทำให้เว็บไซต์ มีชีวิตชีวา น่าสนใจ และเพิ่มประสบการณ์ที่ดีให้กับผู้ใช้ CSS มี 3 เครื่องมือหลัก สำหรับสร้างแอนิเมชัน: Transitions, Transforms, และ Keyframes การเรียนรู้แอนิเมชัน CSS จะช่วยให้คุณสร้าง เว็บไซต์ที่มีปฏิสัมพันธ์ และดึงดูดผู้ใช้งานได้มากขึ้น

CSS Transitions - การเปลี่ยนแปลงแบบนุ่มนวล

Transition ใช้สร้างการเปลี่ยนแปลงที่นุ่มนวลระหว่าง 2 สถานะ Syntax: transition: property duration timing-function delay; เหมาะสำหรับเอฟเฟกต์ง่ายๆ เช่น hover, focus, active
Property ความหมาย ตัวอย่างค่า คำอธิบาย
transition-property property ที่จะเปลี่ยน all, opacity, transform ระบุว่าจะ animate property ใด
transition-duration ระยะเวลา 0.3s, 500ms, 2s ระยะเวลาที่ใช้ในการเปลี่ยน
transition-timing-function ความเร็ว ease, linear, ease-in-out รูปแบบการเปลี่ยนความเร็ว
transition-delay หน่วงเวลา 0s, 100ms, 1s เวลารอก่อนเริ่ม transition
ตัวอย่างการใช้งาน: .button {   background-color: blue;   transition: background-color 0.3s ease; } .button:hover {   background-color: darkblue; }

CSS Transforms - การแปลงรูปแบบ

Transform ใช้เปลี่ยนแปลงรูปร่าง ตำแหน่ง ขนาด และมุมของ element ไม่กระทบต่อ layout ของ elements อื่นๆ มี 4 ฟังก์ชันหลัก: translate, rotate, scale, skew
Transform Function รูปแบบ ตัวอย่าง ผลลัพธ์
translate() translate(x, y) transform: translate(50px, 100px); เลื่อนขวา 50px ลง 100px
rotate() rotate(angle) transform: rotate(45deg); หมุน 45 องศา
scale() scale(x, y) transform: scale(1.5); ขยายใหญ่ขึ้น 1.5 เท่า
skew() skew(x-angle, y-angle) transform: skew(20deg, 10deg); บิดเอียงตามมุมที่กำหนด
การใช้หลาย Transform: transform: translate(100px, 0) rotate(45deg) scale(1.2);

CSS Keyframes - แอนิเมชันที่ซับซ้อน

@keyframes ใช้สร้างแอนิเมชันที่ซับซ้อน กำหนดหลายขั้นตอนได้ สามารถควบคุมทุกขั้นตอนของแอนิเมชันได้อย่างละเอียด รูปแบบ: กำหนดสถานะที่ % ต่างๆ หรือใช้ from/to
รูปแบบ Keyframes ตัวอย่างโค้ด คำอธิบาย
From-To @keyframes slide {
  from { left: 0; }
  to { left: 100px; }
} 		เหมาะสำหรับแอนิเมชัน 2 สถานะ
Percentage @keyframes bounce {
  0% { top: 0; }
  50% { top: 100px; }
  100% { top: 0; }
} 		ควบคุมทุกขั้นตอนได้
Multiple Properties @keyframes colorChange {
  0% { background: red; opacity: 1; }
  100% { background: blue; opacity: 0.5; }
} 		เปลี่ยนหลาย property พร้อมกัน
การเรียกใช้งาน: .box {   animation: slide 2s ease-in-out infinite; }

Animation Properties - ควบคุมแอนิเมชัน

Property ค่าที่ใช้ได้ ตัวอย่าง คำอธิบาย
animation-name ชื่อ keyframes animation-name: slide; ระบุ keyframes ที่จะใช้
animation-duration เวลา (s, ms) animation-duration: 2s; ระยะเวลาแอนิเมชัน
animation-iteration-count จำนวน, infinite animation-iteration-count: 3; จำนวนรอบที่เล่น
animation-direction normal, reverse, alternate animation-direction: alternate; ทิศทางการเล่น
animation-fill-mode none, forwards, backwards, both animation-fill-mode: forwards; สถานะก่อน/หลังเล่น
animation-play-state running, paused animation-play-state: paused; หยุดหรือเล่นแอนิเมชัน
Shorthand Animation: animation: name duration timing-function delay iteration-count direction fill-mode; ตัวอย่าง: animation: slide 2s ease-in-out 0.5s infinite alternate forwards;

Timing Functions - ควบคุมความเร็ว

Timing Functions กำหนดความเร็วของแอนิเมชันในแต่ละช่วงเวลา ช่วยให้แอนิเมชันดูเป็นธรรมชาติมากขึ้น
Timing Function ลักษณะ เหมาะสำหรับ Cubic-bezier เทียบเท่า
linear ความเร็วคงที่ การเลื่อนที่สม่ำเสมอ cubic-bezier(0, 0, 1, 1)
ease เริ่มช้า เร็ว แล้วช้า (default) แอนิเมชันทั่วไป cubic-bezier(0.25, 0.1, 0.25, 1)
ease-in เริ่มช้า แล้วเร็วขึ้น Elements หายไป cubic-bezier(0.42, 0, 1, 1)
ease-out เริ่มเร็ว แล้วช้าลง Elements ปรากฏขึ้น cubic-bezier(0, 0, 0.58, 1)
ease-in-out เริ่มช้า เร็วตรงกลาง ช้าท้าย การเปลี่ยนที่นุ่มนวล cubic-bezier(0.42, 0, 0.58, 1)
steps() เปลี่ยนแบบไม่ต่อเนื่อง Sprite animation steps(4, end)
Custom Timing: transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55); ใช้เว็บไซต์ cubic-bezier.com สำหรับสร้าง custom timing

3D Transforms - มิติที่สาม

3D Transforms เพิ่มความลึกและมิติใหม่ให้กับแอนิเมชัน ต้องกำหนด perspective เพื่อให้เห็นเอฟเฟกต์ 3D
3D Transform รูปแบบ ตัวอย่าง ผลลัพธ์
perspective perspective: value; perspective: 1000px; กำหนดจุดมอง (ยิ่งน้อยยิ่งเหมือนอยู่ใกล้)
translateZ() translateZ(value) transform: translateZ(100px); เลื่อนเข้าหา/ออกจากผู้ชม
rotateX() rotateX(angle) transform: rotateX(45deg); หมุนรอบแกน X (พลิกบนล่าง)
rotateY() rotateY(angle) transform: rotateY(180deg); หมุนรอบแกน Y (พลิกซ้ายขวา)
rotateZ() rotateZ(angle) transform: rotateZ(45deg); หมุนบนระนาบ (เหมือน rotate)
ตัวอย่าง Card Flip: .card-container {   perspective: 1000px; } .card {   transform-style: preserve-3d;   transition: transform 0.6s; } .card:hover {   transform: rotateY(180deg); }

เอฟเฟกต์พิเศษ

เอฟเฟกต์ Properties ตัวอย่างโค้ด
Fade In/Out opacity .fade-in {
  animation: fadeIn 1s ease-in;
}
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
Slide In transform: translateX/Y @keyframes slideIn {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}
Bounce transform: translateY @keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-30px); }
}
Pulse transform: scale @keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.1); }
}
Shake transform: translateX @keyframes shake {
  0%, 100% { transform: translateX(0); }
  25% { transform: translateX(-10px); }
  75% { transform: translateX(10px); }
}

เอฟเฟกต์ ตัวอย่างโค้ด คำอธิบาย

  transform: translateY(-3px);

  width: 100%;
} 		เส้นใต้เลื่อนเข้า

Loading Animations - แอนิเมชันโหลด

ประเภท ตัวอย่างโค้ด ผลลัพธ์
Spinner .spinner {
  border: 4px solid #f3f3f3;
  border-top: 4px solid #3498db;
  border-radius: 50%;
  animation: spin 1s linear infinite;
}
@keyframes spin {
  to { transform: rotate(360deg); }
} 		วงกลมหมุน
Dots .dot {
  animation: bounce 1.4s infinite ease-in-out;
}
.dot:nth-child(2) { animation-delay: 0.2s; }
.dot:nth-child(3) { animation-delay: 0.4s; } 		จุดกระโดด 3 จุด
Progress Bar @keyframes progress {
  from { width: 0%; }
  to { width: 100%; }
}
.bar { animation: progress 2s ease-out; } 		แถบความคืบหน้า

Performance Tips - เพิ่มประสิทธิภาพ

หลักการสำคัญ: - ใช้ transform และ opacity สำหรับแอนิเมชัน (GPU accelerated) - หลีกเลี่ยง animate properties ที่ trigger reflow: width, height, margin, padding - ใช้ will-change เพื่อแจ้ง browser ล่วงหน้า - จำกัดจำนวนแอนิเมชันที่เล่นพร้อมกัน - ใช้ @media (prefers-reduced-motion) สำหรับผู้ที่ไม่ชอบแอนิเมชัน
✅ ควรใช้ (เร็ว) ❌ หลีกเลี่ยง (ช้า) เหตุผล
transform: translateX(100px); left: 100px; transform ไม่ trigger reflow
transform: scale(1.2); width: 120%; scale ใช้ GPU
opacity: 0.5; visibility: hidden; opacity เร็วกว่า
will-change: transform; - เตรียม browser ล่วงหน้า
ตัวอย่างการใช้ will-change: .box {   will-change: transform, opacity; } .box:hover {   transform: scale(1.2);   opacity: 0.8; } Reduced Motion: @media (prefers-reduced-motion: reduce) {   * {     animation-duration: 0.01ms !important;     transition-duration: 0.01ms !important;   } }

Best Practices - แนวทางปฏิบัติที่ดี

แนวทางที่ดี: - ใช้แอนิเมชันอย่างมีจุดประสงค์ - ช่วยให้ UI เข้าใจง่ายขึ้น ไม่ใช่เพื่อความสวยงามเพียงอย่างเดียว - เลือกระยะเวลาที่เหมาะสม - แอนิเมชันเล็กๆ ควรเร็ว (150-300ms), แอนิเมชันซับซ้อนค่อนข้างช้า (400-600ms) - ใช้ easing ที่เหมาะสม - ease-out สำหรับ element ที่ปรากฏ, ease-in สำหรับหายไป - คำนึงถึง accessibility - ใช้ prefers-reduced-motion - ทดสอบบนหลาย device - แอนิเมชันอาจช้าบนอุปกรณ์ที่มีประสิทธิภาพต่ำ - ให้ feedback ที่ชัดเจน - ผู้ใช้ควรเข้าใจว่าเกิดอะไรขึ้น - เก็บ keyframes ไว้ใช้ซ้ำ - สร้าง library แอนิเมชันที่ใช้บ่อย ตัวอย่างระยะเวลาที่แนะนำ: /* Micro-interactions */ .button:hover { transition: 150ms; } /* เปลี่ยนหน้า/view */ .modal { animation: 300ms; } /* Onboarding/tutorial */ .intro { animation: 600ms; } Template แอนิเมชันพื้นฐาน: /* Reusable animations */ @keyframes fadeIn {   from { opacity: 0; }   to { opacity: 1; } } @keyframes slideInUp {   from {     transform: translateY(30px);     opacity: 0;   }   to {     transform: translateY(0);     opacity: 1;   } } /* Usage */ .fade { animation: fadeIn 0.3s ease-out; } .slide-up { animation: slideInUp 0.4s ease-out; }
